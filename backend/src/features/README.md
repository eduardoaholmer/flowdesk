# Features

Pasta vazia de propósito na Sprint 1 (Fundação). A partir da Sprint 2, cada feature de domínio
(`auth/`, `workspaces/`, `teams/`, `issues/`, ...) ganha aqui sua própria pasta com
`router.py`, `service.py`, `repository.py`, `schemas.py` e `models.py`, seguindo o padrão
descrito em `CLAUDE.md` §3–§13.

Não criamos as pastas de feature antecipadamente porque estrutura sem conteúdo é
especulação — ver princípio YAGNI em `CLAUDE.md` §1.6.
